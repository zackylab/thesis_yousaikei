\begin{abstract}
　近年，画像処理技術が広く普及し，医療画像や衛星画像分析などで機械学習を用いた複雑な処理が進展している. このような背景から, 画像処理において耐障害性が重要となり, 我々は，Broadway のデータ取り込みとデータ処理機能を画像処理パイプラインとして活用することに着目する. 研究の目的は, Nodeを用いたBroadwayパイプラインを構築し, 耐障害性を向上させた性能評価を行うことですある. 具体的には, BroadwayのSupervisorを使用してプロセスを監視し, Node間のメッセージ通信によってNIF関数のクラッシュが親ノードに波及しないようにする. 評価手法としては, 画像処理おいて, Fault-Injectionを用いた異常終了のシミュレーションやSupervisorの回復速度の測定が行わる. 実験結果として, 安全性では, 実際にメモリ不足などによる異常終了が発生した場合, 再起動ができた. また, NIF 関数の使用時 abort を発生する場合, 起動元の Erlang VM に波及されなかった. 提案されたBroadwayパイプラインは優れた耐障害性と信頼性を示し, 大規模な画像処理タスクへの適応性が向上している. 将来の課題としては, 特にNIF関数のクラッシュに伴うデータの保存や不整合への対処を考える.
\begin{center}
  \textbf{Abstract}
\end{center}
 　In recent years, image processing technology has become widespread, and complex processing using machine learning is progressing in medical images, satellite image analysis, etc. Against this background, fault tolerance has become important in image processing, and we , we focus on utilizing Broadway's data acquisition and data processing functions as an image processing pipeline.The purpose of the research is to construct a Broadway pipeline using Node and evaluate its performance with improved fault tolerance. Specifically, we use Broadway's Supervisor to monitor the process and ensure that a crash in the NIF function does not spread to the parent node through message communication between nodes.As an evaluation method, in image processing, We simulated abnormal termination using Fault-Injection and measured the recovery speed of Supervisor. As a result of the experiment, in terms of safety, it was possible to restart if an abnormal termination actually occurred due to insufficient memory. Also, When an abort occurs when using a NIF function, it does not affect the Erlang VM that started it. The proposed Broadway pipeline shows excellent fault tolerance and reliability, and is adaptable to large-scale image processing tasks. As future issues, we will particularly consider saving data and dealing with inconsistencies caused by NIF function crashes.
\end{abstract}
